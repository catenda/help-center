# Manuel d'export IFC Revit

Ce manuel d'export IFC a pour objectif de fournir un guide à l'utilisateur sur comment exporter l'IFC aux bonnes fins. Un fichier IFC peut rapidement devenir volumineux et lourd quand vous avez beaucoup d'informations à extraire du modèle. C'est pourquoi lors de l'export d'un IFC, vous devez décocher les informations inutiles. Lors du chargement d'un modèle vers Catenda, il n'est pas toujours nécessaire d'avoir beaucoup d'informations et un niveau de détail élevé dans le modèle. Un peu plus loin dans ce manuel, nous reviendrons sur les paramètres que nous recommandons pour rendre le modèle un peu plus petit et un peu plus facile à utiliser. Ici, nous passerons étape par étape la façon la plus appropriée d'exporter un IFC de Revit vers Catenda.

## 1. **Paramètres du projet**

Avant d'exporter, il est important de vous assurer que les GUID de votre projet Revit sont corrects.

`Gérer -> Paramètres -> Informations du projet -> Paramètres IFC`

![Project information IFC Parameters IfcSite GUID IfcBuilding GUID IfcProject GUID](https://downloads.intercomcdn.com/i/o/975614819/ee92de6f2477949e208dac45/image.png?expires=1779991200&signature=488a733e6dce35b3f9f765548a38ebcbb902309625bfa1827447b3e10ff5d1ca&req=fSciEMh6lYBWFb4V1XW4gW4DZHJ8Y%2FMyB4KsohYxe9JI9upp%2BSOrcthV9MYf%0AU5M5bUP8q8Ck5XUUcK86OtHqKw%3D%3D%0A)

Si le GUID est différent d'un export précédent, les objets des exports plus récents ne seront pas correctement liés aux GUID dans les rubriques BCF.

Quand vous créez un nouveau projet, il aura un identifiant unique.

Lors de l'export des IFC depuis Revit pour une utilisation dans Catenda Hub, nous recommandons de ne pas définir le niveau de détail à élevé. ![](https://downloads.intercomcdn.com/i/o/areracg3/1892285324/5ee98db642dda09e9ef2e32f89f3/image.png?expires=1779991200&signature=a1105615a979210c8668e250b9ba318c89bc4fbd6335483a32692b3f7a71552f&req=dSguFMt2mIJdXfMW3nq%2BgeyMDiMPhq7Q85wgYm3Xh1XpOK5XvxcASYTyhduW%0AsLQ%2B1rJyZBFN%2FAQvoW31uHOg%2F7A%3D%0A)

_Exporter les pièces comme éléments de construction_

Exporter les pièces comme élément IFC standard.

_Autoriser l'utilisation de représentation "Modèle solide" mélangée_

Sélectionnez cette option pour autoriser le mélange des géométries BRep et d'extrusion pour une unité.

_Utiliser la vue active lors de la création de géométrie_

Sélectionnez cette option pour utiliser la vue active pour générer la géométrie. Notez que cela peut avoir des résultats inattendus s'il est utilisé sur une vue non-3D.

_Utiliser le nom de la famille et du type comme référence_

Sélectionnez cette option pour utiliser les noms de famille et de type pour les références.

_Utiliser les limites de pièce 2D pour le volume de la pièce_

Sélectionnez cette option pour utiliser une approche simplifiée pour calculer le volume de la pièce (basée sur l'extrusion des limites d'espace 2D) qui est également standard lors de l'export vers IFC 2x2.

_Inclure l'élévation IfcSite dans l'origine du placement local du site_

Sélectionnez cette option pour inclure la hauteur du décalage Z pour la position locale dans IfcSite. Supprimez l'option pour l'exclure.

_Stocker le GUID IFC dans un paramètre d'élément après l'export_

Sélectionnez cette option pour enregistrer les GUID IFC générés dans le fichier du projet après l'export. Cela ajoutera des paramètres "GUID IFC" aux éléments et à leurs types ainsi qu'aux informations du projet pour les guides du projet, du site web et de la construction.

_Exporter la boîte de délimitation_

Sélectionnez cette option pour exporter les représentations "Boîte de délimitation". Cette option reste sélectionnée automatiquement pour l'export GSA.

_Conserver la géométrie tessellée comme triangulation_

Si vous avez des éléments courbes complexes ou des shells et qu'ils ne s'affichent pas correctement après l'export IFC, vous pouvez sélectionner cette option. Gardez à l'esprit que vous pourriez produire un fichier IFC très lourd.

_Utiliser le nom du type uniquement pour le nom du type IFC_
