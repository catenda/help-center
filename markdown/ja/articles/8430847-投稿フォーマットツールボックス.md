# 投稿フォーマットツールボックス

> 投稿を作成できるフィールドをフォーマットする方法

同じフォーマットルールがCatenda Hub全体のさまざまな投稿に適用されます。

投稿には、topic説明、コメント、承認説明、および承認コメントを含めることができます。

投稿を編集できるユーザーへのアクセス権によって異なる場合があります。

Catendaはマークダウンを使用してテキストをフォーマットします。

つまり、文の前後の特定の文字がテキストのスタイルに影響を与えます。

これがtopicヘッダーと説明の送信後の見た目です:

![](https://downloads.intercomcdn.com/i/o/areracg3/2142853925/ac356cf8aa82ab2d579f021fbb5a/image.png?expires=1779991200&signature=db379701aa782a795d0715249d2e131838f324a1adc1df5c4f091de05493201d&req=diEjFMF7nohdXPMW3nq%2BgeO5Gu4QTVbwiyiagSAGJYRM3jg%2FjBE06qW2yxJo%0Ah8KhRqQ6FpjyJBQAessFwOYLWgY%3D%0A)

## 1. フォーマット利用可能性

フィールドが編集されるときにフィールドの下に表示されるツールボックスにより、送信されたフィールドがフォーマット可能であることがわかることが多くあります。

> \_\_注:\_\_ブラウザのズームスケールを十分に縮小して、すべてのツールを確認してください。 フォーマットは以下のフィールドで利用可能です:

### 1.1 **Topic説明とコメント**

既存のtopicのコメントを編集するか、新しいtopicを送信するときのいずれかで、ツールボックスを表示するために説明またはコメントを編集します:

![](https://downloads.intercomcdn.com/i/o/areracg3/2142716774/e00776f86b3e5961148459c33a7b/image.png?expires=1779991200&signature=0e22d9a262f6ec8cc9ffd59b884be7ed21d8b5d927709293d35243a523fd2783&req=diEjFM5%2Fm4ZYXfMW3nq%2BgbCbYvmyhkqzs2jDl1gilk2Cc6xR6QmhVhC4KlhS%0ACdo6DkIGE0Ixwn%2FSMNbyKzybwrM%3D%0A)

### 1.2 **承認リクエスト説明**

新しい承認リクエストダイアログの説明にはツールボックスが表示されませんが、フォーマットはこの説明に適用されます。

### 3.3 **リンク**

リンクスタイル設定は、行の任意の部分で実行でき、行の開始時に実行する必要がある他のスタイル設定と一緒に効果的に機能します。

説明内のリンクは緑色で下線が引かれます。

リンクのテキストは、リンクが指すurlと異なる場合があります。

URLは自動的にクリック可能なリンクに変換されます

```
https://hub.catenda.com
```

になります

[https://hub.catenda.com](https://hub.catenda.com)

そして

```
[Catenda Hub](https://hub.catenda.com)
```

になります:

[Catenda Hub](https://hub.catenda.com)

> \_\_注:\_\_リンクをクリックするときは、[括弧内の]テキストは1つのリンクを示しているかもしれませんが、(括弧内の)実際のリンクが異なる場合がある可能性があるため、注意してください。 リンクをクリックする前に、リンク上でマウスをかざして、ブラウザがどこにリダイレクトするかを確認することをお勧めします。

### 3.4 **フォーマット設定を防止**

文字列以外の文字を単語で囲むことで、単語を別の外観にすることができます。

これは常に望ましいわけではありません。

`\\`をそのような文字の前に置くと、`\\`は消えます。

これらの文字に適用されたフォーマット設定は機能しなくなります。

`\\`を[リスト](#h_6da4949f8c)の一部ではない空行に置くと、結果は空行のままになります。

### 4.2 **リスト**

_順序なしリスト_

順序なしリストは、その上に空行で始まり、その後、ハイフン(`-`)、プラス(`+`)、またはアスターリスク(`\*`)を入力した後にスペースを入力します。

行の開始時に4つのスペースまたはタブスペースを挿入して、サブリストを作成します。

```
 - 最初の要素 - 2番目の要素     - 行の開始時に4つのスペースを挿入してサブリストを作成します。
```

または

```
 + 最初の要素 + 2番目の要素     + 行の開始時に4つのスペースを挿入してサブリストを作成します。
```

または

```
 * 最初の要素 * 2番目の要素     * 行の開始時に4つのスペースを挿入してサブリストを作成します。
```

すべてがこれになります:

- 最初の要素
- 2番目の要素
  - 行の開始時に4つのスペースを挿入してサブリストを作成します。

> \_\_注:\_\_順序なしリストが正しくフォーマットされるには、リストの上の行に空の改行が必要です。 _順序付きリスト_ 空行を上に置いた順序付きリストを開始し、数字、ピリオド、スペース（`1. `） 前の数字が何であるかは重要ではなく、数字の後にピリオドが続くことが重要です。 行の最初に4つのスペースまたはタブスペースを挿入してサブリストを作成します。

```
1. アイテム1
2. アイテム2
3. アイテム3
    4. アイテム4
```

または

```
1. アイテム1
1. アイテム2
1. アイテム3
    1. アイテム4
```

または

```
1. アイテム1
10. アイテム2
1. アイテム3
    1000. アイテム4
```

すべてこのようになります：

1. アイテム1
2. アイテム2
3. アイテム3
   4. アイテム4

> **注：** 順序付きリストが正しくフォーマットされるには、リストの上の行に空行が必要です。 開始する数字は番号付けが始まる場所に影響しません

```
23. 23
1. 24
1. 25
    1. 26
```

このようになります：

1. 23
2. 24
3. 25
   4. 26
   同じ投稿内の2番目のリストで番号をリセットして1から開始したい場合は、その間に空行または[区切り](#h_3a36cfbc61)を入れることができます。
   [リストがフォーマットされないようにする](#h_2ec17c688b)には、独自の番号付けを使用できます：
   リストの上の行に文字を配置します。 良い選択は`\\`です。消えるからです。

```
\23. 23
24. 24
25. 25
    26. 26
```

ピリオド（`.`）の前にバックスラッシュ（`\\`）を付けます：

```
23\. 23
24\. 24
25\. 25
    26\. 26
```

すべてこのようになります：

23. 23
24. 24
25. 25
26. 26

## 6. **@ メンバーとチームを言及する**

`@`ツールをクリックするか、説明またはコメント内に`@`を書いて、メンバーを言及します。

`@`を入力すると、メンバーとチームのドロップダウンリストが表示されます。

![](https://downloads.intercomcdn.com/i/o/areracg3/2102724602/dfd63733158f932a304596decf84/image.png?expires=1779991200&signature=266a7840bf95b8b253ec385b2f655a8f49bdec2f03f681715a2184b5321bc158&req=diEnFM58mYdfW%2FMW3nq%2BgQgVJSLkSxRSadTEPadiZdNpnB391hNKmHZ5LqXx%0AJCj3xsiGglM4P1a7sLfHtZH61EM%3D%0A)

メールアドレス、メンバー名、またはチーム名を入力してリストを検索します。

矢印キーでリストを上下に移動し、クリックするか、Enterキーを押してメンバーまたはチームを選択します。

メンバーまたはチームがこのリストに表示されるには、メンバーまたはチームが少なくともトピックボードへの読み取りアクセス権を持っている必要があります。

リストでメンバーまたはチームを選択した後、`@`は次のようなテキストを取得します：

`@[\<email address of member>]`または`@[\<Team name>]`

### 6.1 **トピックの言及を保存または送信する**

説明が保存されるか、コメントが送信されると、トピックボードへのアクセス権を持つ関連メンバーが通知を受け取ります。

メンバーのメールアドレスまたはチームの名前が既知の場合、手動で書くこともできますが、トピックボードの一部でない場合、関連メンバーは彼らが言及されていることを通知されません。

_メンバー言及時の通知_

トピックボードへのアクセス権を持ち、言及されているメンバーは、トピックで言及されていることを通知します。

_チーム言及時の通知_

トピックボードへのアクセス権を持ち、言及されているチームの一部であるメンバーは、彼らが属するチームが言及されていることを通知します。

_将来のトピックイベントについての通知_

トピック説明とコメント内で言及されているチームのメンバーは、保存された説明または送信されたコメントで1つの通知のみを受け取ります。

言及されていることの通知に加えて、投稿で言及されているメンバーは、自動的に[フォロー](https://support.catenda.com/en/articles/8053299-right-panel-in-an-issue#h_a97f3b264e)に設定されます。トピックは新しいコメントとステータス変更などのトピックイベントについて通知を受け取ります。

これは、[担当者](https://support.catenda.com/en/articles/8400566-issue-header#h_0a91fa8dd9)と[リクエスト者](https://support.catenda.com/en/articles/8400566-issue-header#h_1aea0990a7)だけでなく、トピックの将来の変更について通知を受け取るようにするための優れた方法です。

メンバーがトピックのフォローを希望しなくなった場合、手動でフォロー解除する必要があります。

### 6.2 **投稿での言及**

投稿での言及は、緑色のテキスト色を持つことで識別できます。

![](https://downloads.intercomcdn.com/i/o/areracg3/2102770866/23a219c6303ae2c3ea3167ea8839/image.png?expires=1779991200&signature=dc54372f275beb9d6b9f58eef6364046025ccb05f18518fc1e8b56576f8e78ec&req=diEnFM55nYlZX%2FMW3nq%2Bgc6BCXnvU1Bkq3F79VNCulbUfUkNi0%2FhK1QXHGmC%0AYCoFipDW17p1vsYnR4usRrYZ%2FUo%3D%0A)

バックグラウンドでは、言及されたプロジェクト参加者の一意のIDが保存されます。

この緑色のテキストに表示されるのは参加者の名前です。

メンバーまたはチームが名前を変更しても、投稿で言及されたままですが、新しい名前です。

言及されているメンバーには、そのメンバーの[メンバーページ](https://support.catenda.com/en/articles/8228836-member-page)に向かうクリック可能なリンクがあります。

言及されているチームには、そのチームの[チームページ](https://support.catenda.com/en/articles/7891755-team-page)に向かうクリック可能なリンクがあります。

_存在しないメンバー_

言及されているメールアドレスを持つプロジェクト内のメンバーがない場合、投稿は代わりに次のようになります：

![](https://downloads.intercomcdn.com/i/o/areracg3/2102805445/b2e3443410ea96885a2d85676dee/image.png?expires=1779991200&signature=d64e4fdacb46dba5067b67f2b53dcb3371157a4c792de007bdcd5790b6455038&req=diEnFMF%2BmIVbXPMW3nq%2BgV2%2BX4QvdVlKUmPY0wchlEm3gW%2FQgNf9nrFfqLi1%0ASRk1H9bn2Dppn%2B4cO%2F3q%2Bp1QwFE%3D%0A)

これは、メールが間違ってフォーマットされているか、メンバーがプロジェクトの一部ではなくなったことが原因である可能性があります。

このメールアドレスを持つメンバーが将来プロジェクトの一部になる場合、投稿はそのメンバーの名前を表示するように変更されます。

_存在しないチーム_

チームがプロジェクトから削除され、新しいチームが作成された場合、この新しいチームは言及されません。

新しいチームを言及するには、投稿を再提出する必要があります。

## 7. **# タグ付きトピック**

`#`ツールをクリックするか、説明またはコメント内に`#`を書いてトピックをタグ付けします。

`#`を入力すると、すべてのトピックボードからのトピックのドロップダウンリストが表示されます。

![](https://downloads.intercomcdn.com/i/o/areracg3/2102803413/752177c9a293de34a6bd9c7c4d36/image.png?expires=1779991200&signature=2e3d8eaa05e4d86071987625f7277fe7bd34ade7ba287ffe84d17863c5b1e2fd&req=diEnFMF%2BnoVeWvMW3nq%2BgQslLQuJRz%2F96TrjXxebVMDqy6wbx57JHhESfLh5%0Aq4PCAx7ujDa%2BJIEGQaJVDZ%2BZpiU%3D%0A)

タイトルまたはトピック番号を入力してリストを検索します。

矢印キーでリストを上下に移動し、クリックするか、Enterキーを押してトピックを選択します。

トピックがこのリストに表示されるには、タグ付けするメンバーが、トピックがあるボードにアクセスできる必要があります。

リストでトピックを選択した後、`#`は次のようなテキストを取得します：

`#[\<topic number>]`

### 7.1 **タグ付きトピックを保存または送信する**

説明が保存されるか、コメントが送信されると、トピック関係が作成されます。

リンクされたトピックは、トピックの[右側のメニュー](https://support.catenda.com/en/articles/8053299-right-panel-in-an-issue)の[リンクされたトピック](https://support.catenda.com/en/articles/8053299-right-panel-in-an-issue#h_edf2cb07a8)のリストに追加されます。

![](https://downloads.intercomcdn.com/i/o/areracg3/2103668127/63b222278014acedd5e109eaa00f/image.png?expires=1779991200&signature=2149815ea61b5a424e16434a30602bf0ad7bdfac5716a9be69b7fc5329c0ca7c&req=diEnFc94lYBdXvMW3nq%2BgUcCxcZ9fv6bg1pEo4juADZYfI3MQkCannAOnEBB%0AleI3JRHXDIbKQfJ1rhkXj6yPxJo%3D%0A)

別のトピックがタグ付けされたトピックに加えて、タグ付けされたトピック自体が、それが追加された[右側のメニュー](https://support.catenda.com/en/articles/8053299-right-panel-in-an-issue)の[リンクされたトピック](https://support.catenda.com/en/articles/8053299-right-panel-in-an-issue#h_edf2cb07a8)のリストにトピックへのリンクを受け取ります。

![](https://downloads.intercomcdn.com/i/o/areracg3/2103660685/9068edb07ddc704d58d821a0aa4c/image.png?expires=1779991200&signature=b3cb88b489a1e13e91119acb5a1e65869e6137d08aa613a34673a44e9d6145d3&req=diEnFc94nYdXXPMW3nq%2BgZtQFZ6%2FadIHexDqOuXukfxVutzxFMdv4Csh2%2BBJ%0AJ4PgbRdgBEqA%2F0O6%2Bv%2FB4%2BrLgiA%3D%3D%0A)

トピックリンクは、説明にタグ付きトピックがあるトピックから、またはタグ付けされたトピックから、各トピックにアクセスして[右側のメニュー](https://support.catenda.com/en/articles/8053299-right-panel-in-an-issue)の[リンクされたトピック](https://support.catenda.com/en/articles/8053299-right-panel-in-an-issue#h_edf2cb07a8)のリストを編集することで、後で削除できます。

### 7.2 **投稿でのタグ付きトピック**

タグ付きトピックは、最初にそのステータスの名前とともにタグ付きトピックの現在のステータスの色の円を持つことで識別できます。

その後、トピックタイトルが表示され、その後にトピック番号が続きます。

チェックボックスとともに、閉じられたタグ付きトピックは上部に表示されるプログレスバーのプログレスに向かってカウントされ、開いているタグ付きトピックはプログレスがカウントされる総アイテムに向かってカウントされます。

![](https://downloads.intercomcdn.com/i/o/areracg3/2103628989/5fbf0cd8c9301753582aea8950e9/image.png?expires=1779991200&signature=ca04d06ea11db334d5d09d8015a562933dff76c96ee77f0067fcc66830a84137&req=diEnFc98lYhXUPMW3nq%2BgW6UTt%2FkO1pC00sooBVCn30sIjAm6dD8c3fkOEkR%0ArVc8ifPy8Jud7vIGWaSPdO6o7iY%3D%0A)

タグ付きトピックには、タグ付きトピックをそのトピックボードで開くクリック可能なリンクがあります。
