# 投稿フォーマットツールボックス

> 投稿を作成できるフィールドをフォーマットする方法

Catenda Hub全体のさまざまな投稿に対して、同じフォーマットルールが適用されます。投稿には、トピックの説明、コメント承認の説明、および承認コメントを含めることができます。投稿の編集が可能なユーザーのアクセス権が異なる場合があります。Catendaはマークダウンを使用してテキストをフォーマットします。これは、文の前後の特定の文字がテキストのスタイルに影響することを意味します。これは、トピックのヘッダーと説明がどのように見えるかの例です。提出後：

![](https://raw.githubusercontent.com/catenda/help-center/main/images/tt9y4c9z/01-intro.png)

## 1. フォーマット可用性

フィールドが編集される際にフィールドの下に表示されるツールボックスにより、送信されたフィールドをフォーマットできることがしばしば分かります。

> \*\*注：\*\*ブラウザのズームスケールを十分にズームアウトして、すべてのツールを表示します。

フォーマットは以下のフィールドで利用可能です。

### 1.1 **トピックの説明とコメント**

既存のトピックの説明またはコメントを編集するか、新しいトピックを送信するときに、ツールボックスを表示します。

![](https://raw.githubusercontent.com/catenda/help-center/main/images/tt9y4c9z/02-topic-description-and-comment.png)

### 1.2 **承認リクエストの説明**

ツールボックスは新しい承認リクエストダイアログの説明に表示されていませんが、フォーマットはこの説明に適用されます。

![](https://raw.githubusercontent.com/catenda/help-center/main/images/tt9y4c9z/03-approval-request-description.png)

> \*\*注：\*\*このフィールドは、承認リクエストを送信した後は編集できません。

### 1.3 トピックボードの説明

ツールボックスは新しいトピックボードの説明に表示されていますが、ボードが送信されるとき説明がフォーマットされないことに注意することが重要です。

![](https://raw.githubusercontent.com/catenda/help-center/main/images/tt9y4c9z/04-topic-board-description.png)

## 2. **トピックコメント添付**

トピックコメントでは、添付ツールが表示されます。トピックの+ボタンをクリックして、添付ファイルを追加します。

![](https://raw.githubusercontent.com/catenda/help-center/main/images/tt9y4c9z/05-topic-comment-attachment.png)

コメント添付の詳細については、[ここ](https://support.catenda.com/en/articles/8053352-topic-body-the-content-of-a-topic#h_0e3c0059ae)をクリックしてください。

## 3. **テキストのスタイル設定**

次の方法を使用すると、テキストにスタイルを付けることができます。

### 3.1 **太字、斜体、打消し線**

太字、斜体、打消し線のスタイル設定は行の任意の部分で行うことができ、行の開始時に配置する必要があるその他のスタイル設定とうまく機能します。

### 3.2 **ヘッダー**

ヘッダーツールは、行の開始にハッシュタグ/ポンド記号を適用します。5つのレベルのヘッダーがサポートされています。ヘッダーが受け取るフォーマットは、行の開始時のハッシュタグ/ポンド記号（`#`）の量に依存します。

<img alt="" src="https://raw.githubusercontent.com/catenda/help-center/main/images/tt9y4c9z/inline-c21079ec7efb.png" width="290"/>   \<->   <img alt="" src="https://raw.githubusercontent.com/catenda/help-center/main/images/tt9y4c9z/inline-557784a5d702.png" width="290"/>

### 3.3 **リンク**

リンクのスタイル設定は行の任意の部分で行うことができ、行の開始時に配置する必要があるその他のスタイル設定とうまく機能します。説明内のリンクは緑色で下線が引かれます。テキストは、リンクが指すURLと異なる場合があります。URLは自動的にクリック可能なリンクに変わります。

```
https://hub.catenda.com
```

[https://hub.catenda.com](https://hub.catenda.com)になり、

```
[Catenda Hub](https://hub.catenda.com)
```

以下のようになります。[Catenda Hub](https://hub.catenda.com)

> \*\*注：\*\*リンクをクリックする際は注意してください。括弧からのテキスト[括弧内]はある場合でも1つのリンクを示す可能性がありますが、実際のリンク（括弧から）は異なる可能性があります。 ブラウザがリダイレクトする場所を確認し、リンクをクリックする前にリンクにカーソルを合わせることをお勧めします。

### 3.4 **フォーマット防止**

数字以外の文字で単語をラップすることで、それらの外観を変えることができます。これは常に望ましいわけではありません。そのような文字の前に`\`を置くと、`\`は消えます。これらの文字に適用されたフォーマットは機能しなくなります。空行に`\`を置き、[リスト](#lists)の一部ではない場合、結果は空行のままになります。

## 4. **テキストの分割と構造の追加**

次の方法を使用すると、テキストを分割して構造を追加することで、テキストを強化できます。

### 4.1 **画像**

説明の画像にリンクを付けることができます。画像を埋め込むには、この構文を使用できます。

```
![text](https://bimsync.com/img/favicon/dark-mode/favicon-32x32.png)
```

この画像を取得するには、「text」という単語が付いています。

![text](https://raw.githubusercontent.com/catenda/help-center/main/images/tt9y4c9z/06-images.png)

画像が正しく表示されるには、Catendaが提供されたリンクにアクセスできる必要があります。

> **注：** \+ 画像を説明に追加できます。\+ 画像をテキストと混合できるため、同じコメントまたは説明で画像の前後の両方にテキストを持つことができます。 \+ このような画像は、テーブルおよびリストと組み合わせることができます。 たとえば、フォーマットチェックボックスの代わりに、文/テーブルの中央に小切手画像を追加できます。これは、文の開始時にのみ機能します。\- リンクがリンク画像が将来利用可能であると保証することはできません。

_Catendaイメージドキュメントリンクの取得_右クリック[ドキュメント](https://support.catenda.com/en/articles/4670288-actions-in-the-document-structure#h_133e2bcc57)を選択した後のダウンロードアクションボタンまたは右クリック[ダウンロード最新リビジョンアクションボタン。ドキュメント](https://support.catenda.com/en/articles/9323521-actions-in-a-document)を開いた後、ドキュメントのダウンロードリンクをコピーできます。上記のようなトピックでこのリンクを使用すると、Catendaドキュメントをトピックで使用できます。

> **注：** \+ Catendaイメージドキュメントへのリンクを追加すると、Catendaにのみ表示される画像を追加でき、トピックが交換される可能性のある他のプラットフォームでは利用できません。 \+ Catendaイメージドキュメントへのリンクを追加することにより、ドキュメントにアクセスできる人だけが画像を見ることができます。 \- Catendaイメージドキュメントへのリンクを追加すると、トピックが同期された可能性のある他のプラットフォームに表示されず、ドキュメントにアクセスできない人は表示されません。

### 4.2 **リスト**

**順序なしリスト**空白行でリストを開始して、ハイフン（`-`）、プラス（`+`）、またはアスタリスク（`*`）のいずれかを続けてスペースを挿入します。行の開始時に4つのスペースまたはタブスペースを挿入して、サブリストを作成します。

```
 - First element - Second element     - Insert 4 spaces at the start of the line to create a sublist.
```

または

```
 + First element + Second element     + Insert 4 spaces at the start of the line to create a sublist.
```

または

```
 * First element * Second element     * Insert 4 spaces at the start of the line to create a sublist.
```

これはすべてこれに変わります。

- First element
- Second element
  - Insert 4 spaces at the start of the line to create a sublist.

> \*\*注：\*\*順序なしリストを正しくフォーマットするには、リストの上の行に空白の改行が必要です。

**順序付きリスト**空白行でリストを開始して、数字、ピリオド、スペース（`1. `)前の数字が何であるかは重要ではなく、ピリオドが続く数字であるだけです。行の開始時に4つのスペースまたはタブスペースを挿入して、サブリストを作成します。

```
1. item one 2. item two 3. item three     4. item four
```

または

```
1. item one 1. item two 1. item three     1. item four
```

または

```
1. item one 10. item two 1. item three     1000. item four
```

これはすべてこれに変わります。

1. item one
2. item two
3. item three

4. Item four

> \*\*注：\*\*順序付きリストを正しくフォーマットするには、リストの上の行に空白の改行が必要です。

開始する番号は、番号付けが始まる場所に影響しません。

```
23. twenty-three 1. twenty-four 1. twenty-five     1. twenty-six
```

これに変わります。

1. twenty-three
2. twenty-four
3. twenty-five

4. twenty-six

同じ投稿内の2番目のリストで番号付けをリセットして1から始めたい場合は、その間に空白または[分割線](#dividers)を置くことができます。

独自の番号付けを使用できるように、[リストがフォーマットされないようにする](#preventing-formatting)場合は、リストの上の行に文字を配置します。良い選択は`\`です。消えてしまいます。

```
\23. twenty-three 24. twenty-four 25. twenty-five     26. twenty-six
```

ピリオド（`.`）の前に逆スラッシュ（`\`）を置きます。

```
23\. twenty-three 24\. twenty-four 25\. twenty-five     26\. twenty-six
```

これはすべてこれに変わります。

23\. twenty-three 24\. twenty-four 25\. twenty-five 26\. twenty-six

### 4.3 **分割線**

3つ以上のダッシュ`---`が独自の行に表示される分割線を作成します。

---

### 4.4 **チェックリスト**

これらはトピックの説明で確認できます。_アクセス権限：_トピックボードへのアクセス権を記述します。

```
- [ ] first item - [x] second item     - [ ] sublist item
```

これはこれに変わります。

![](https://raw.githubusercontent.com/catenda/help-center/main/images/tt9y4c9z/07-checklists.png)

プロジェクトでチェックリストを使用する方法に関する簡単なビデオを見るには、[ここ](https://support.catenda.com/en/articles/5036461-check-lists-within-issues)をクリックしてください。`- [ ]`または`- [x]`と入力した場合、コメントが送信されるか編集後に保存されると、チェックボックスはチェックなしとチェック済みの状態のままになります。

**コメント内のチェックボックス**コメント内のコメント、チェックボックスはフォーマットによってのみチェックできます。コメント内のチェックボックスをクリックしてチェックすることはできません。_アクセス権限が必要-_コメント作成者が作成されたコメントの編集にアクセスできます。

## 5. **@メンバーとチームの言及**

`@`ツールをクリックするか、説明またはコメントに`@`を書いてメンバーに言及します。`@`を入力した後、メンバーとチームのドロップダウンリストが表示されます。

![](https://raw.githubusercontent.com/catenda/help-center/main/images/tt9y4c9z/10-mentioned-members-and-teams.png)

メール、メンバー名、またはチーム名の入力を開始して、リストを検索します。矢印キーを使用してリストを上下に移動し、クリックするか、Enterキーを押してメンバーまたはチームを選択します。メンバーまたはチームがこのリストに表示されるには、メンバーまたはチームがトピックボードへの読み取りアクセス権を持つ必要があります。リストからメンバーまたはチームを選択した後、`@`はいくつかの追加テキストを取得します。

`@[<email address of member>]`または`@[<Team name>]`

### 5.1 **トピック内の言及の保存または送信**

説明が保存されるか、コメントが送信されると、トピックボードにアクセスできる関連メンバーが通知を受け取ります。メンバーのメールまたはチームの名前が既知の場合は、手動で書くこともできますが、トピックボードの一部ではない場合、関連するメンバーは言及されたことについて通知されません。

**メンバー言及時の通知**トピックボードにアクセスできるメンバーが言及されている場合、トピックに言及されたという通知を受け取ります。

**チームメンション時の通知**トピックボードにアクセスでき、言及されるチームの一部であるメンバーは、彼らが一部であるチームがトピックで言及されたという通知を受け取ります。

**将来のトピックイベントに関する通知**トピックの説明とコメント内で言及されているチームのメンバーは、保存された説明または送信されたコメント後のみ1つの通知を受け取ります。

言及されたことの通知に加えて、投稿で言及されるメンバーは自動的に[follow](https://support.catenda.com/en/articles/8053299-right-panel-in-an-issue#h_a97f3b264e)トピックに設定され、新しいコメントやステータス変更などのトピックイベントについての通知を取得します。これは、[受託者](https://support.catenda.com/en/articles/8400566-issue-header#h_0a91fa8dd9)と[リクエスト者](https://support.catenda.com/en/articles/8400566-issue-header#h_1aea0990a7)だけが、トピック上の将来の変更について通知を取得することを確認する優れた方法です。メンバーがトピックの追跡を続けたくない場合は、手動でそれをはずす必要があります。

### 5.2 **投稿内の言及**

投稿内の言及は、緑色のテキスト色を持つことによって識別できます。

![](https://raw.githubusercontent.com/catenda/help-center/main/images/tt9y4c9z/11-mention-in-post.png)

背景では、言及されたプロジェクト参加者の一意のIDが保存されます。この緑色のテキストで表示される参加者の名前です。メンバーまたはチームが名前を変更した場合でも、投稿で言及されたままですが、新しい名前の下に留まります。

言及されたメンバーには、そのメンバーの[メンバーページ](https://support.catenda.com/en/articles/8228836-member-page)に向けられるクリック可能なリンクがあります。言及されたチームには、そのチームの[チームページ](https://support.catenda.com/en/articles/7891755-team-page)に向けられるクリック可能なリンクがあります。

**非実在メンバー**プロジェクト内に、言及されたメールアドレスを持つメンバーがない場合、投稿は次のように見えます。

![](https://raw.githubusercontent.com/catenda/help-center/main/images/tt9y4c9z/12-mention-in-post.png)

これは、メールが誤ってフォーマットされたか、メンバーがプロジェクトの一部でなくなったためです。このメールアドレスを持つメンバーが将来プロジェクトの一部になった場合、投稿は変更されてそのメンバーの名前を表示します。

**存在しないチーム**チームがプロジェクトから削除され、新しいチームが作成された場合、この新しいチームは言及されません。新しいチームに言及するには、投稿を再送信する必要があります。

## 6. **#タグ付きトピック**

`#`ツールをクリックするか、説明またはコメントに`#`を書いてトピックをタグ付けします。`#`を入力した後、すべてのトピックボードからトピックのドロップダウンリストが表示されます。

![](https://raw.githubusercontent.com/catenda/help-center/main/images/tt9y4c9z/13-tagged-topics.png)

タイトルまたはトピック番号の入力を開始して、リストを検索します。矢印キーを使用してリストを上下に移動し、クリックするか、Enterキーを押してトピックを選択します。リストにトピックが表示されるには、タグ付けしているメンバーがトピックが入っているボードへのアクセス権を持つ必要があります。リストからトピックを選択した後、`#`はいくつかの追加テキストを取得します。

`#[<topic number>]`

### 6.1 **タグ付きトピックの保存または送信**

説明が保存されるか、コメントが送信されるとき、トピック関連が作成されます。リンクされたトピックは、トピックの[右メニュー](https://support.catenda.com/en/articles/8053299-right-panel-in-an-issue)内の[リンクされたトピック](https://support.catenda.com/en/articles/8053299-right-panel-in-an-issue#h_edf2cb07a8)リストに追加されます。

![](https://raw.githubusercontent.com/catenda/help-center/main/images/tt9y4c9z/14-saving-or-submitting-a-tagged-topic.png)

別のトピックがタグ付けされたトピックに加えて、タグ付けされたトピック自体は、トピックの[右メニュー](https://support.catenda.com/en/articles/8053299-right-panel-in-an-issue)の[リンクされたトピック](https://support.catenda.com/en/articles/8053299-right-panel-in-an-issue#h_edf2cb07a8)リストに追加されたトピックへのリンクバックを受け取ります。

![](https://raw.githubusercontent.com/catenda/help-center/main/images/tt9y4c9z/15-saving-or-submitting-a-tagged-topic.png)

トピックリンクは、説明でタグ付きトピックを持つトピックから削除するか、タグ付けされたトピックから、各トピックに移動して[右メニュー](https://support.catenda.com/en/articles/8053299-right-panel-in-an-issue)で[リンクされたトピック](https://support.catenda.com/en/articles/8053299-right-panel-in-an-issue#h_edf2cb07a8)リストを編集することによって削除できます。

### 6.2 **投稿内のタグ付きトピック**

タグ付きトピックは、まず現在のステータスのタグ付きトピックの色の円を持つことで識別できます。そのステータスの名前と一緒に。その後、トピックのタイトルはトピック番号が続きます。

チェックボックスとともに、閉じられているタグ付きトピックは、上部に向かって表示される進捗バーの進捗に向かってカウントされ、タグ付きトピックが開いているときは、進捗がカウントされる総項目に向かってカウントされます。

![](https://raw.githubusercontent.com/catenda/help-center/main/images/tt9y4c9z/16-tagged-topic-in-post.png)

タグ付きトピックには、タグ付きトピックをそのトピックボードで開くクリック可能なリンクがあります。
