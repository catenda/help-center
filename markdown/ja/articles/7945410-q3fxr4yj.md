# zip / ファイル構造をアップロードする

通常の zip ファイルのアップロードと比較して、upload a zip 関数は zip を解凍します。この方法により、手動でフォルダーを作成しなくても、ファイル構造を[ドキュメント ページ](https://support.catenda.com/en/articles/8204673-documents-page#h_28610e1d21)にインポートできます。フォルダーがすでに存在する場合、ファイルは既存のフォルダーに保存されます。

ドキュメントがすでに存在する場合、ファイルはそのドキュメントの新しいリビジョンになります。

## 2. **ファイルサイズ制限の回避**

zip をアップロードすると、zip によってファイルが圧縮されるため、7 GB を超えるファイルをアップロードできます。

## 4. **デスクトップコネクタ**

[Catenda Desktop Connector を](https://support.catenda.com/en/articles/8358861-catenda-desktop-connector)使用すると、ドキュメントの最新バージョンをローカル システムから Catenda Hub に自動的かつ定期的にアップロードできます。

Desktop Connector は、ドキュメントを 1 回の大きなドラッグ アンド ドロップや zip アップロード バッチでアップロードするのではなく、ファイルごとにアップロードするため、通常のアップロード プロセスよりも処理が速く、失敗のリスクも最小限に抑えられます。
