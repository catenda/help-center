# IFC Export Archicad マニュアル

IFC をエクスポートする場合、IFC エクスポートに何が関連しているかに注意してください。IFC ファイルは情報が多く含まれている場合、大きくなる可能性があり、扱いが難しくなります。したがって、不要な情報をエクスポートしないことが重要です。このレポートでは、Archicad で IFC エクスポートをフィルターするためのさまざまなヒントを提供します。

## 1. プロジェクト情報

IFC をプロジェクトからエクスポートする前に、プロジェクト情報が設定されていることを確認してください。プロジェクト情報は以下の場所にあります。

`ファイル -> 情報 -> プロジェクト情報`

プロジェクト情報は以下のようになります。

![](https://raw.githubusercontent.com/catenda/help-center/main/images/tjawkzdp/01-project-info.png)

デフォルトでは、さまざまなフィールドは空になっています。空の場合でも、リストされている各プロジェクト、サイト、建物に対して一意の ID が作成されます。状況によっては、同じ Archicad プロジェクトから複数の IFC ファイルがエクスポートされます。その例を以下に示します。

異なる研究分野からのオブジェクトを含む IFC ファイル。ダクト付きの MEP モデル、壁付きのアーキテクチャモデル、すべてのスラブ付きの構造モデル。

複数の建物 複数の建物が同じ Archicad ファイル内でモデル化されることもあり、それぞれについて個別の IFC がエクスポートされます。

異なる設計上の選択。同じ Archicad ファイルで建物のさまざまなバリエーションがモデル化されている場合、各バリエーションは多くの場合、独自の IFC ファイルにエクスポートされます。

### 1.1 プロジェクト情報を設定する

複数の IFC ファイルをエクスポートする予定があるかどうかに関わらず、後で変更する必要がないようにプロジェクト情報に値を入力することは多くの場合、良い考えです。プロジェクト情報はファイルの GUID に影響を与えるため、記入することが重要です。エクスポートされる IFC の各タイプは、独自のプロジェクト設定を持つ必要があります。プロジェクト設定は右下にインポートおよびエクスポートできます。この方法で、ファイルからのエクスポートに対して複数のプロファイルをサポートできます。Catenda では、異なるモデルの IFC が異なる情報で構成されていることが重要である一方、同じモデルにある IFC は同じ情報で構成されていることが重要です。

## 2. IFC エクスポート設定

プロジェクト全体をエクスポートするには。3D ビューに立つ必要があります。正しいトランスレーターを使用していることを確認してください。

![](https://raw.githubusercontent.com/catenda/help-center/main/images/tjawkzdp/02-ifc-export-settings.png)

オプションボタンをクリックして、IFC エクスポート設定の概要を確認します。モデルフィルターの下では、さらにフィルターアウトするものを選択することもできます。

![](https://raw.githubusercontent.com/catenda/help-center/main/images/tjawkzdp/03-ifc-export-settings.png)

IFC トランスレーターボックスを使用すると、トランスレーター設定を表示または変更したり、新しいトランスレーターを作成したりできます。

IFC エクスポートを変更する場合は、デフォルトのトランスレーターを台無しにしないよう、作成済みのトランスレーターの 1 つを複製することをお勧めします。トランスレーターをここに複製します。

新規作成 > 複製 > 複製するトランスレーターを選択 をクリックします。

IFC を結合する場合は、同じバナーの下でそれを行うことができます。

![image-2.png](https://raw.githubusercontent.com/catenda/help-center/main/images/tjawkzdp/04-ifc-export-settings.png)

### 2.1 エクスポートのさまざまな設定

1\. [モデルフィルター](#model-filter)を使用すると、異なるプリセットによってエクスポートする内容をフィルターアウトできます。2\. [タイプマッピング](#type-mapping)を使用すると、各要素がエクスポートされる IFC タイプを選択できます。3\. [ジオメトリ変換](#geometry-conversion)を使用すると、エクスポートするジオメトリの種類を選択できます。4\. [プロパティマッピング](#property-mapping)を使用すると、タイプに基づいて基準を設定できます。5\. [データ変換](#data-conversion)を使用すると、モデルからエクスポートするデータの種類を選択できます。6\. [単位変換](#unit-conversion)を使用すると、IFC でエクスポートする測定単位を選択できます。

## 4. タイプマッピング

IFC がエクスポートされると、モデルのすべての要素に IFC タイプが割り当てられます。

使用する IFC トランスレーターを選択したら、タイプマッピングに移動して「インポート用に IFC タイプをマップ」をクリックして、エクスポートされた IFC で必要なタイプマッピングの種類を管理できます。

![](https://raw.githubusercontent.com/catenda/help-center/main/images/tjawkzdp/07-2-type-mapping.png)

タイプマッピングには主に、IFC タイプを並べ替えるための 2 つの異なるオプションがあります。

### 4.1 要素タイプ

各要素には、基本的な IFC タイプが自動的に割り当てられます。プロジェクトマネージャーと要素設定で各要素の割り当てられた IFC タイプを確認できます。

### 4.2 分類

この方法は、分類基準を指定に従って、より柔軟で詳細な IFC タイプマッピングを可能にします。ゾーンと開口要素は固定の IFC タイプに設定されています。IFCSpace および IFCOpeningElements。

![](https://raw.githubusercontent.com/catenda/help-center/main/images/tjawkzdp/08-classification.png)

## 5. ジオメトリ変換

IFC エクスポート用のジオメトリ変換では、ジオメトリをさまざまな方法で変換できます。

![](https://raw.githubusercontent.com/catenda/help-center/main/images/tjawkzdp/09-3-geometry-conversion.png)

### 5.1 Archicad 要素の変換

**すべてのモデル要素のジオメトリをエクスポート:** _例外付きのパラメトリック_ - デフォルト _BREP_ - 正確なジオメトリ - これが選択されている場合、次の 2 つのオプションも BREP になります

**ソリッド要素操作内の要素:** _押し出し/回転_ - デフォルト _BREP_ - 正確なジオメトリ

接合部を持つ要素 一部の要素は他の要素と交差して、押し出しの一部をトリミングできます。_押し出し/回転_ - 要素の角を接続する このオプションは押し出され要素にジャンクションを追加します。このオプションを使用すると、特に傾斜した壁や屋根などの要素がうまく接続されることがわかります。

_接合部なしの押し出し/回転_ - デフォルト - これにより、エクスポートが高速化されます 90 度の要素（壁や屋根など）がある場合、このオプションで違いが気付かないでしょう。傾斜した壁がある場合、壁が接続する角で、直線押し出しの過剰をトリミングする代わりに、壁は互いに真っ直ぐに過ぎ去り、うまく接続されていないことに気付くでしょう。

_BREP_ - 正確なジオメトリ

パラメトリック要素 Archicad 要素は、パラメトリック ジオメトリとしてエクスポートでき、多くの場合、距離まで押し出されるガイド ラインの形式でエクスポートできます。パラメトリック ジオメトリにより、作成後にオブジェクトを簡単に編集できます。このオプションは編集性を保持していますが、定義はジオメトリの近似であるため、ジオメトリはしばしばやや不正確になる場合があります。IFC を Archicad または別の編集プログラムに戻すインポートして、さらに編集を加える予定がある場合、このオプションをお勧めします。

BREP 別の BREP サーフェスとしてジオメトリをエクスポートします。長さ/幅/高さなどのパラメータで、生成されたサーフェスでオブジェクトを説明する代わりに、各サーフェスは 3D 内の場所を持つ個別のサーフェスとして説明されます。オブジェクトは複数のサーフェスを含むことができますが、これらは緩いサーフェスになります。BREP をエクスポートした後、パラメータがサーフェスに適用されなくなるため、サーフェスを個別に編集する必要があります。BREP オプションを使用すると、正確なサーフェスがエクスポートされ、オーサリング ツール間のパラメータの解釈に違いの余地がありません。サーフェスは個別に色付けされ、Catenda Hub では Archicad と同じ色になります。サーフェスの色は、次のメニューで設定できます。

`オプション -> 要素属性 -> サーフェス`

サーフェスのすべての設定を設定することは、IFC に書き込まれるため、良い考えかもしれません。

![](https://raw.githubusercontent.com/catenda/help-center/main/images/tjawkzdp/10-conversions-of-archicad-elements.png)

Catenda 3D ビューアのサーフェスは、光源がない場合、フラット シェーディングを備えています。つまり、アンビエント、減衰、光沢、エミッション、スペキュラなどの値は、Catenda 3D ビューアでサーフェスを表示する際に解釈されません。以下の値は、Catenda が 3D ビューアでサーフェスを表示する際に解釈します。

サーフェスカラー サーフェスの色

透過性 オブジェクトがどの程度見えるか (半透明の不透明度設定をオーバーライドします)

拡散 サーフェスの色がどの程度暗いまたは明るいか

**IFC モデル位置を定義:**このオプションはプロジェクトの位置を決定します。最初からプロジェクト内の参照用の共通座標ポイントに同意することが重要です。これを変更すると、複数のコンサルタントがオブジェクトを移動する必要があることを意味する場合があり、これはプロジェクトの終盤では多くの場合実行不可能です。

測量ポイントとプロジェクトの原点 - デフォルト このオプションを使用すると、Archicad のモデルまで (0,0,0) からの距離は、Catenda Hub のモデルまで (0,0,0 + 調査ポイントの座標) からの距離 + プロジェクトの場所を設定している場合、モデルは Catenda Hub のそれらの座標で終了します

プロジェクトの原点のみ このオプションを使用すると、Archicad のモデルまで (0,0,0) からの距離は、Catenda Hub のモデルまで (0,0,0) からの距離になります。

測量ポイントのみ このオプションを使用すると、Archicad の測量ポイントからモデルまでの距離は、Catenda Hub のモデルまで (0,0,0) からの距離になります

### 5.2 IFC スキーマ関連オプション

**材料保存モード (IFC2x3 のみ)** - 要素を分割しない、保存は保証されません このオプションを使用すると、オブジェクト全体を 1 つのオブジェクトとしてエクスポートします

- 必要な場合のみ分割して材料を保存 - デフォルト
  このオプションを使用すると、必要な場合にのみ各複合材料の個別オブジェクトをエクスポートします

- すべての要素を部品に分割し、材料を保存
  このオプションを使用すると、すべてのオブジェクトが各材料の個別オブジェクトとしてエクスポートされます

複合壁材がある場合、各材料について個別のオブジェクトがエクスポートされます。通常、1 ~ 5mm の蒸気バリアがあり、非常に薄いオブジェクトになります。オブジェクトがその程度薄い場合、ジオメトリは不正確になる可能性があります。問題のある壁の素材は次のようなものです。

![](https://raw.githubusercontent.com/catenda/help-center/main/images/tjawkzdp/11-ifc-schema-related-options.png)

これにより、不正確さがため穴が正確にどこにあるべきかをピンポイントするのが難しくなるため、ソフトウェアがサーフェスを通してホールをカットするのが困難になる可能性があります。したがって、開口部がカットされていないことに気付いた場合は、このオプションをオンにすることが役に立つ場合があります。このオプションをオンにすると、壁が多くの薄い複合オブジェクトの代わりに 1 つの単一オブジェクトとしてエクスポートされます。

### 5.3 複合構造と複雑なプロファイル

**複合建築要素を部品に分割**ここでは、複合要素を分割する要素の種類と、分割しない要素の種類を選択できます。このオプションを選択すると、材料の保存モードを選択できなくなります。

## 6. プロパティマッピング

プロパティマッピング内 (ファイル > IFC > 相互運用性 > プロパティマッピング) でエクスポートする IFC のバージョンを選択できます。標準 IFC2x3 と標準 IFC4 があります。IFC でエクスポートするために psets を追加することもできます。これを行う場合は、選択する IFC スキーマの複製を作成する必要があります。

![](https://raw.githubusercontent.com/catenda/help-center/main/images/tjawkzdp/12-4-property-mapping.png)

標準 IFC の複製を作成したら、IFC スキーマを選択して _エクスポート用の IFC プロパティをマップ_ をクリックして、その新しいプリセットに必要なプロパティを追加できます。

![](https://raw.githubusercontent.com/catenda/help-center/main/images/tjawkzdp/13-4-property-mapping.png)

---

## 7. データ変換

データ変換の下で、IFC エクスポートから取得する、ジオメトリに加えてどのような種類のデータを選択します。エクスポートする内容のボックスをチェックします。

要素パラメーター Archicad 要素パラメーターを読み取り、IFC 数量または IFC プロパティに変換します。タイプに応じて。このオプションを選択することで、ファイルサイズが大幅に増加します。

![](https://raw.githubusercontent.com/catenda/help-center/main/images/tjawkzdp/14-5-data-conversion.png)

IFC 基本数量 サイズ、面積、体積のパラメーターを読み取ります。このボックスをオフにしない場合、IFC を Catenda Hub にインポートするのに問題が生じる可能性があります。

## 8. 単位変換

エクスポートの長さ、角度、面積、体積、通貨、時間単位を設定します。
