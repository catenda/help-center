# チームの作成

建設プロジェクトでは、人員の出入りや役割の変更、異なる作業グループへの異動が頻繁に発生するため、チームは非常に価値があります。チームを設定することで、プロジェクトで役割が変更された後の誤解を防ぐことができます。

[メンバーとチームページ](https://support.catenda.com/en/articles/4670291-members-and-teams-page)の緑色のプラスボタンの右側にあるアクションメニューを開いてチームを作成します。選択されているタブに関係なく作成できます。

![](https://raw.githubusercontent.com/catenda/help-center/main/images/dwahhbcw/01-intro.png)

チームは、メンバーとチームのコンテンツページからも作成できます。

## 1. **チームをいつ設定する?**

Catendasのチームはどのような方法でも分割できます。通常、チームはプロジェクト参加者が属する各専門分野ごとに構成されます。その専門分野内の個別作業グループまたは学際的なチームなど、別のチームが作成される場合があります。メンバーは複数のチームに属することができるため、属するチームはプロジェクトでの異なる役割を表します。

### 1.1 **トピック管理用のチーム**

トピックはチームに要求し割り当てることができるため、チームの責務と要求を簡単に追跡できます。新しく作成されたトピックはまずチームに割り当てられることが多く、その後チームのオープンなトピックのレビューが実施され、トピックはフォローアップのために個別メンバーに更に割り当てられます。

### 1.2 **トピック通知用のチーム**

トピック内では、チームを言及して、チーム全体に何かを知らせることもできます。

アクセス制御はチームに委譲することができます。これを使用すると、ユーザーがチームの一部になる前にアクセスを設定できます。その後、アクセスを使用する準備ができたら、適切なユーザーをチームに追加できます。これの良い使用例は、メンバーが招待されたときにチームを設定することで、プロジェクトの一部になるとすぐに適切なアクセスを得られるようにすることです。

## 2. **新しいチームを作成する**

新しいチームの作成ページは次のようになります。

![](https://raw.githubusercontent.com/catenda/help-center/main/images/dwahhbcw/02-create-a-new-team.png)

### 2.1 **名前**

ここにチームの名前を入力します。チーム名はプロジェクトごとに一意である必要があります。

### 2.2 **役割**

役割領域で、このチームの一部となるメンバーが持つべき役割を設定できます。

**管理者** - チームが管理者役割を取得する場合、そのチームのすべてのメンバーは、メンバーとして管理者であるかどうかに関わらず、管理者アクセスを持ちます。管理者はプロジェクトへの完全なアクセス権を持ちますが、いくつかの例外があります。- プロジェクトの削除は、プロジェクトオーナーのみが実行できます。- プライベートブックマーク、コレクション、プライベートライブラリ設定などのユーザー固有のビュー。

_メンバー_ - デフォルトでは、メンバーはほとんどの機能に対してデフォルトで読み取りと書き込みアクセス、そして自分のコンテンツに対して削除アクセスを持ちます。メンバーは、ドキュメント、トピックボード、その他のアプリの異なるセクションでのアクセス制御ルールによって、さらに制限される可能性があります。

### 2.3 **作成**

チームが作成されると、[チームコンテンツページ](https://support.catenda.com/en/articles/7891755-team-page)が開きます。そこからメンバーをチームに追加できます。
