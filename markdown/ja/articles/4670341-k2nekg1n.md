# Revit IFC エクスポート マニュアル

この IFC エクスポート マニュアルの目的は、適切な目的で IFC をエクスポートする方法についてのガイドをユーザーに提供することです。モデルから取得する情報が多い場合、IFC ファイルはすぐに重くなり、大きくなる可能性があります。そのため、IFC をエクスポートする際には、不要な情報のチェックを外す必要があります。モデルを Catenda にアップロードするとき、モデルに多くの情報と高レベルの詳細が含まれている必要は必ずしもありません。このマニュアルの少し後では、モデルを少し小さくして作業しやすくするために推奨される設定に戻ります。ここでは、IFC を Revit から Catenda にエクスポートする最も適切な方法を段階的に説明します。

この記事では、以下に関する情報を見つけることができます。

## 1. **プロジェクト設定**

エクスポートする前に、Revit プロジェクトの GUID が正しいことを確認することが重要です。

新しいプロジェクトを作成すると、一意の ID が割り当てられます。

3. アクティブなビューを使用してジオメトリを生成するには、このオプションを選択します。これを非 3D ビューで使用すると、予期しない結果が生じる可能性があることに注意してください。

2D 空間境界の押し出し)、これも IFC 2x2 にエクスポートするときの標準です。

10. BAT-ID またはオブジェクトの ID をエンティティの名前として表示する場合は、このオプションを選択します。

### 3.6 **地理的参照**

Revit の座標がプロジェクト内の他のモデルと同期され、最終的に同じ場所になることが重要です。

したがって、点メジャーを使用して Catenda Hub の座標を測定し、Catenda Hub で測定した点と同じ場所にある点で Revit の座標基準を指定します。

![](https://downloads.intercomcdn.com/i/o/969764101/0c12135b4cfe78982de2d2c9/image.png?expires=1781092800&signature=84d865b4e05cd94c652262461fda5186cf3f7d946257cdddba401888aea06abd&req=fSYuEc96nIFeFb4V1XW4gXGcH6FISuvCuGsiUzSJvpN%2B2gT%2BbnAigyT02RI%2F%0AlYpLb4rMYF4tiqRbu5bmBqCC2g%3D%3D%0A)

このオプションは、[管理] タブ -> [座標] -> [座標ベースの指定] にあります。_点の座標_を指定します

北/南、東/西、標高の座標を指定して、モデルを再配置し、モデルを真北に回転します。

Revit では、多くの場合、90 度の角度でモデリングする方が簡単で、モデル全体を回転する必要はありません。この場合、代わりに真北を回転させることができます。このオプションは、「管理」タブの「座標」の下にある「位置」ドロップダウンにあります。
