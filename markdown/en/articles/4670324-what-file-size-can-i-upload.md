## **Splitting into smaller files**

Large .ifc and .ifczip files are recommended to be divided into smaller files. Splitting can help adhere to the 750 MB upload limitation and allows for more manageable file handling within projects.

By uploading several smaller files it both becomes easier for project members to turn on and off different parts and to download separate parts of the file individually.

## **Exporting to IfcZip**

Some authoring tools support IfcZIP as an option when exporting IFC files.

> **Note:** **Example:** `3D-model.ifczip`

## **Creating an IfcZip**

If the size of a `.ifc` exceeds the upper limit, a `.ifczip` file can be created to remain eligible for upload. Follow these steps to turn a `.ifc` into a `.ifczip`:

- Compress the `.ifc` file to a `.zip` file.
- Change the file extension from `.zip` to `.ifczip`

> **Note:** **Example:** `3D-model.ifc` v `3D-model.zip` v `3D-model.ifczip ​`

## **Contact support**

If altering the file is not an option, and the file is larger than what can be uploaded, please let us know at [support@catenda.com](mailto:support@catenda.com). Files are handled on a case by case basis. Before reaching out, consider using alternative methods like creating an `.ifczip`, using the Desktop Connector, or splitting the file into smaller parts.
