# Revisions in naming convention

With the naming convention, it is requested or desired that everything is uniquely named, checked and then uploaded. However, blocks such as the date or the revision are constantly changing. So how to handle this usecase? There are two options:

1. Keep each document as a single document

    <div class="intercom-container"><img height="910" src="https://raw.githubusercontent.com/catenda/help-center/main/images/jzbw5hey/inline-8d2926a56c34.png" style="height: auto;" width="1920"/></div>

1. Switch off" the Date block in the naming convention. This works in the Document recognition column. This block is then not included in the query. It is then also not displayed in the name (document area).

    <div class="intercom-container"><img height="910" src="https://raw.githubusercontent.com/catenda/help-center/main/images/jzbw5hey/inline-a2fc3b4afcfe.png" style="height: auto;" width="1920"/></div>

    To do this, you can display the "Name de Revision" column, which shows the original name.

    <div class="intercom-container"><img height="910" src="https://raw.githubusercontent.com/catenda/help-center/main/images/jzbw5hey/inline-1b1cee11c3c8.png" style="height: auto;" width="1920"/></div>
