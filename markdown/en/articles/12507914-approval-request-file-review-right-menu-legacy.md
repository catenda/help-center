# Approval request file review right menu - Legacy

When the [file review of an approval](https://support.catenda.com/en/articles/12507422-file-review-in-an-approval-request-legacy) is first opened its right menu might be closed. Click on the information button to the right to expand this menu.

![](https://raw.githubusercontent.com/catenda/help-center/main/images/zt2p953n/01-intro.png)

This is what the right menu in the file review of an approval can look like for projects that enabled shared revisions before 2 October 2025:

![](https://raw.githubusercontent.com/catenda/help-center/main/images/zt2p953n/02-intro.png)

## 1. **File overview**

### 1.1 **Document**

The documents page file path and document name of the document. If the document has a latest published revision it is displayed with:

- Revision name
- Publishing member
- File size
- Publish date

### 1.2 **File for approval**

The file that is currently being reviewed with:

- Revision name
- Publishing member
- File size
- Publish date

### 1.3 **Validation preview**

The status of the validation is displayed in the priview. When not all reviewers have submitted their validation it will be yellow and say pending. When a reviewer submits their validation the bar for that member will highlight either green or red depending on theri validation.

When a reviewer has submitted a rejection, the box will change to red and say rejected.

![](https://raw.githubusercontent.com/catenda/help-center/main/images/zt2p953n/03-validation-preview.png)

When the minimum amount of members as specified in the approval have submitted approval validations the box will change to green and say approved.

![](https://raw.githubusercontent.com/catenda/help-center/main/images/zt2p953n/04-validation-preview.png)

**Hover** Hover over the validation preview box to see who which reviewer is involed in which action and if they submitted their validation, when they submitted it.

![](https://raw.githubusercontent.com/catenda/help-center/main/images/zt2p953n/05-validation-preview.png)

## 2. **Comment**

### 2.1 **Comment box**

Enter a comment to add a comment to the review of this file. Comments follow the general rules of [formatting of posts](https://support.catenda.com/en/articles/8430847-formatting-of-posts) on Catenda. Validations can be submitted without entering a comment.

### 2.2 **Comment**

Click on comment to submit a comment without rejecting or approving.

### 2.3 **Reject**

Click reject to submit a rejection along with the comment that is written in the comment box.

### 2.4 **Approve**

Click reject to submit a rejection along with the comment that is written in the comment box.

**Approve with comment** Open the approve action menu and click approve with comment to apply the approve with comment status wich can have effect on how the document is published. This option can only be chosen if something is written in the comment box.
