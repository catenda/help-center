# How to use custom fields in naming convention

In order to use the naming convention with custom fields, custom fields should first be made available on the [custom fields page](https://support.catenda.com/en/articles/6550459-custom-fields-page) so they can be selected in the naming convention configuration. _Access required:_ Project adminstrator

To use a custom field in a naming convention follow these steps:

## 1. **Go to the naming convention content page**

Click on [settings](https://support.catenda.com/en/articles/4670273-project-settings-page) in the left navigation menu to open find the sub-pages to project settings. Click on Naming convention that now shows up as a sub-page to settings to open the [namig convention page](https://support.catenda.com/en/articles/7832559-naming-conventions-page). Click on one of the naming conventions to open the naming convention content page.

![Select naming conventions to customize](https://raw.githubusercontent.com/catenda/help-center/main/images/s9hns6p2/01-go-to-the-naming-convention-content-page.png)

## 2. **Edit naming convention**

On the naming convention conten page, edit the naming convention by clicking on the edit button below the descripton and abov the able with blocks.

![edit your naming convention by clicking on ](https://raw.githubusercontent.com/catenda/help-center/main/images/s9hns6p2/02-edit-naming-convention.png)

## 3. **Select a pre-existing custom field as source**

In the source column of the naming convention, choose one of the custom fields that are available in the project. In this example, the custom field "**NC Phase**" is selected. This is a dropdown custom fields with values like: Plan, Dsg, EXE...

![choose the custom fields created beforehand in the source section while editing a custom field](https://raw.githubusercontent.com/catenda/help-center/main/images/s9hns6p2/03-select-a-pre-existing-custom-field-as-source.png)

It is advised to use only custom fields in the "**Source**" section, in this example all these custom fields are of type "**drop down list**" or "**Integer**" for the "**Version**" block

![Only use custom fields in the source section](https://raw.githubusercontent.com/catenda/help-center/main/images/s9hns6p2/04-select-a-pre-existing-custom-field-as-source.png)

Documents uploaded to a folder where this naming convention is configured will now be allowed to have values in the names based on the selected custom field.
