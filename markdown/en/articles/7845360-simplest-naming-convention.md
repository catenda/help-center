# Simplest naming convention

This article will show what a [naming convention](https://support.catenda.com/en/articles/7832559-naming-conventions-page) can look like in the [documents section](https://support.catenda.com/en/articles/4670292-documents-page). Before you start reading through this article it is important that you understand the basics of [naming conventions](https://support.catenda.com/en/articles/7832559-naming-conventions-page). You can find out how to use [custom fields](https://support.catenda.com/en/articles/6550459-custom-fields-setup), [set up a naming convention](https://support.catenda.com/en/articles/7832559-naming-conventions-page) and [assign the convention](https://support.catenda.com/en/articles/7831371-document-settings#h_6672c14d90) to a folder. First I would like to start out with the [simplest convention possible](#h_667e7d43ef) to explain what this [naming convention](https://support.catenda.com/en/articles/7832559-naming-conventions-page) does when applied to a folder. Later in the article we can work our way to more complex conventions.

The following topics will be described in this article:

[Folders and conventions](http://Folders%20and%20conventions) - [Uploading a document](#h_5a4a3f35ef) - [Documents and conventions](#h_eb4ed69f2b)

## 1. **Filename.Extension**

Let's start off with the simplest convention possible to see what a convention can look like when it is enabled. This way you will know how to recognize when a convention is active. The simplest convention possible is made by making a new convention and clicking submit right away:

![](https://raw.githubusercontent.com/catenda/help-center/main/images/c55e7gpw/01-filename-extension.png)

This convention will accept files of any name and with any extension. Although this is no different than the regular upload process this is what is done in this article to show what a convention can look like.

_Applying the [naming convention](https://support.catenda.com/en/articles/7832559-naming-conventions-page) to a folder_ [Naming conventions](https://support.catenda.com/en/articles/7832559-naming-conventions-page) are turned on per folder in [document settings](https://support.catenda.com/en/articles/7831371-document-settings#h_6672c14d90). After applying this convention to a folder you will notice that it will have a gear icon.

![](https://raw.githubusercontent.com/catenda/help-center/main/images/c55e7gpw/02-filename-extension.png)

**Information panel:** If you select the folder you will also notice a change in the information panel on the right. Here the [preview](https://support.catenda.com/en/articles/7832559-naming-conventions-page#h_c2f7e24fbc) of the expected document name will appear under the name of the folder. As always you can hover over the [preview](https://support.catenda.com/en/articles/7832559-naming-conventions-page#h_c2f7e24fbc) to see information about the different blocks.

### 1.1 Folders and conventions

When you make a new folder in a convention folder you will see that it has a gray gear badge. This means that it has inherited a [naming convention](https://support.catenda.com/en/articles/7832559-naming-conventions-page) from a parent folder.

![](https://raw.githubusercontent.com/catenda/help-center/main/images/c55e7gpw/03-folders-and-conventions.png)

If you select a folder with a [naming convention](https://support.catenda.com/en/articles/7832559-naming-conventions-page) applied you will see a naming convention [preview](https://support.catenda.com/en/articles/7832559-naming-conventions-page#h_c2f7e24fbc) under the name of the folder in the information panel on the right. This [preview](https://support.catenda.com/en/articles/7832559-naming-conventions-page#h_c2f7e24fbc) depicts the document name pattern that new documents in this folder have to follow.

![](https://raw.githubusercontent.com/catenda/help-center/main/images/c55e7gpw/04-folders-and-conventions.png)

### 1.2 Uploading a document

When you upload a file to a folder that has a [naming convention](https://support.catenda.com/en/articles/7832559-naming-conventions-page) applied you will see that it either gets accepted or rejected depending on the criteria of the [naming convention](https://support.catenda.com/en/articles/7832559-naming-conventions-page). In the example from above both the name and the extension of the file can have any characters of any length.

Below you will see what it looks like when a file with the name test.txt is uploaded:

![](https://raw.githubusercontent.com/catenda/help-center/main/images/c55e7gpw/05-uploading-a-document.png)

_[Block preview](https://support.catenda.com/en/articles/7832559-naming-conventions-page#h_c2f7e24fbc)_ As you can see in the gray area below Upload document in addition to the name of the folder you are uploading your document to you will now also see a [preview](https://support.catenda.com/en/articles/7832559-naming-conventions-page#h_c2f7e24fbc) of what file name the [naming convention](https://support.catenda.com/en/articles/7832559-naming-conventions-page) expects. If you hover over any of the blocks in the [preview](https://support.catenda.com/en/articles/7832559-naming-conventions-page#h_c2f7e24fbc) you will see they are document blocks expecting text of a variable length.

![](https://raw.githubusercontent.com/catenda/help-center/main/images/c55e7gpw/06-uploading-a-document.png)

You can now click on upload and the file will be uploaded.

### 1.3 Documents and conventions

In a folder where a [naming convention](https://support.catenda.com/en/articles/7832559-naming-conventions-page) is applied you will see that documents have a gear badge on their icon. This badge means that the document exists in a folder with a [naming convention](https://support.catenda.com/en/articles/7832559-naming-conventions-page) applied. It is possible that the document was uploaded before the [naming convention](https://support.catenda.com/en/articles/7832559-naming-conventions-page) was applied so it does mean that it follows the naming rules.

![](https://raw.githubusercontent.com/catenda/help-center/main/images/c55e7gpw/07-documents-and-conventions.png)

When you open the document to view it, the first thing you will see is that the document name of is grayed out. Document names in a folder with a [naming convention](https://support.catenda.com/en/articles/7832559-naming-conventions-page) follow the rules of the convention. It is therefore not possible to edit the name of such a document.

![](https://raw.githubusercontent.com/catenda/help-center/main/images/c55e7gpw/08-documents-and-conventions.png)

_This document does not comply with the [naming convention](https://support.catenda.com/en/articles/7832559-naming-conventions-page)_ If a the [naming convention](https://support.catenda.com/en/articles/7832559-naming-conventions-page) is applied to a folder that already had documents in it there is a chance the name of the revision does not comply with the convention. You will see that this is the case when there is an explanation mark behind it.

![](https://raw.githubusercontent.com/catenda/help-center/main/images/c55e7gpw/09-documents-and-conventions.png)

This can be because of two reasons.

1. The revision file name does not match the expected file name of the [naming convention](https://support.catenda.com/en/articles/7832559-naming-conventions-page).
1. The revision file name does not match the expected document outcome according to the [naming convention](https://support.catenda.com/en/articles/7832559-naming-conventions-page).
