# File review in an approval request - Legacy

The file review for an approval request can be opened by clicking on review or view on the approval request page. This is what the review page of a file in an approval request can look like for projects that enabled shared revisions before 2 October 2025:

![](https://raw.githubusercontent.com/catenda/help-center/main/images/3vfvkbhy/01-intro.png)

## 1. **Opening the file review page**

### 1.1 **All documents**

Click on the review button towards the top right of an approval to start reviewing the first document in the approval. Access required: Reviewer or publisher

### 1.2 **Selected documents**

Click on the action menu for a revision and click on add review to review that specific revision. Access required: Reviewer or publisher

### 1.3 **View only**

Click on row in a table on the approval page to open the file review for that specific document.

## 2. **Breadcumbs**

### 2.1 **Documents**

Click on documents to go to the documents page

### 2.2 **Approvals**

Click on Approvals to go to the approvals page

### 2.3 **Approval number**

The number of the approval Click on the number of the approval request to go the the approval request page of the approval request that this revision is part of.

## 3. **File review navigation**

### 3.1 **Back to button**

Go back to the approval request page that files under review are part of.

### 3.2 **Progress bar**

Shows the revisions that can be navigated to

> **Note:** It is only possible to navigate to as many revisions as were selected on the approval requets page. Clicking review on the overview tab of the approval request will let you navigate to all revisions while selecting documents in the documents tab of the approval request and clicking review in the right menu will only let you navigate to the selected documents.

This total number is therefore not always indicative of the total files to review in the approval request.

_Previous and next buttons_ - Go to the previous or next revision in the approval request review.

## 4. **Right menu**

The right menu can be opened by clicking on the info icon towards the right of the approval request review page. The Info icon can look something like this:

![](https://raw.githubusercontent.com/catenda/help-center/main/images/3vfvkbhy/02-right-menu.png)

Click here to read more about the right menu on the approval request review page.

## 5. **Fullscreen**

The review fullscreen button can be found towards the top right of the approval request review page:

![](https://raw.githubusercontent.com/catenda/help-center/main/images/3vfvkbhy/03-fullscreen.png)

## 6. **Document overview**

### 6.1 **Title**

The revision name of the revision that is being reviewed.

### 6.2 **Validation preview**

The status of the validation is displayed in the priview. When not all reviewers have submitted their validation it will be yellow and say pending. When a reviewer submits their validation the bar for that member will highlight either green or red depending on theri validation.

When a reviewer has submitted a rejection, the box will change to red and say rejected.

![](https://raw.githubusercontent.com/catenda/help-center/main/images/3vfvkbhy/04-validation-preview.png)

When the minimum amount of members as specified in the approval have submitted approval validations the box will change to green and say approved.

![](https://raw.githubusercontent.com/catenda/help-center/main/images/3vfvkbhy/05-validation-preview.png)

**Hover** Hover over the validation preview box to see who which reviewer is involed in which action and if they submitted their validation, when they submitted it.

![](https://raw.githubusercontent.com/catenda/help-center/main/images/3vfvkbhy/06-validation-preview.png)

### 6.3 **Document preview**

The document preview on the approval request review page works the same as the [document preview](https://support.catenda.com/en/articles/5001165-document-preview-and-annotations) when opening a document from the documents table.
