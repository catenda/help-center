# Approval settings page - Legacy

The approval settings page can be found by clicking the approval settings action in the action menu to the right of the green plus button on the [approvals page](https://support.catenda.com/en/articles/8349340-approvals-page). _Access required_: Administrator

This is what the banner on the approval settings page can look something like this:

![](https://raw.githubusercontent.com/catenda/help-center/main/images/eekvwhj0/01-intro.png)

## 1. **Actions on the approvals settings page**

The approval settings page actions can be found by clicking on the action menu on the top right of the approvals settings page. This is what the approvals settings page actions can look like:

![](https://raw.githubusercontent.com/catenda/help-center/main/images/eekvwhj0/02-actions-on-the-approvals-settings-page.png)

Click [here](https://support.catenda.com/en/articles/12367730-topic-templates-page) to read more about the topic templates page.

## 2. **Publish options**

Select default behavior that will occur when a published revision gets created in a document as a result of a final validation in an approval. This is what the approval settings page can look like:

![](https://raw.githubusercontent.com/catenda/help-center/main/images/eekvwhj0/03-publish-options.png)

This is what the approval settings page can look like after published statuses for each outcome have been created and configured as well as topic templates:

![](https://raw.githubusercontent.com/catenda/help-center/main/images/eekvwhj0/04-publish-options.png)

### 2.1 **File review status**

The following approval validation outcomes can make different topic creation behavior occur upon final validation:

- Approved
- Approved with comment
- Pending approval
- Rejected

### 2.2 **Document status**

Click on the document status dropdown to select a status from the list of published statuses in the project or set no status.

![](https://raw.githubusercontent.com/catenda/help-center/main/images/eekvwhj0/05-document-status.png)

To edit the statuses in the project go to the status workflow menu of the [document settings page](https://support.catenda.com/en/articles/7831371-document-settings-page).

### 2.3 **Topic template**

When a topic template is configured a topic can be created for each published revisions that is created as a result of a final validation in an approval request. If a markup is created on a draft revision during the approval process one comment is added in the created topic for each set of annotations in the markup.

![](https://raw.githubusercontent.com/catenda/help-center/main/images/eekvwhj0/06-topic-template.png)

**Do not create topics / Do not save markups** By default the setting is set to do not create topics which it can be set it back to by selecting do not save markups in the dropdown.

**Create a new topic template** Click [here](https://support.catenda.com/en/articles/12372007-creating-a-new-topic-template) to read more about creating a topic template.

**Topic preview** If a topic template is selected click on the eye next to the template to see the contents of that topic template.

![](https://raw.githubusercontent.com/catenda/help-center/main/images/eekvwhj0/07-topic-template.png)

### 2.4 **Save**

Click on save to save the changes

## 3. **New validation workflow**

The approvals settings page is not available for projects starting out with the new validation workflow
