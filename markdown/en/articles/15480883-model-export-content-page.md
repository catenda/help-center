# Model Export Content Page

Click on the card of an export in the exports list on the [model exports page](https://support.catenda.com/en/articles/4670280-model-export) to go to the content page for that export. This is what the model export content page can look like:

![](https://raw.githubusercontent.com/catenda/help-center/main/images/3ajw00qp/01-intro.png)

## 1. **Download**

Click on the download button towards the top right of the export to start downloading the zip file of the export. The zip will include each of the models listed as included for this export below.

> **Note:** The current set of models in for each export can be different from when the export was created depending on if each of the models still exists in the project or if any of the revisions have been withdrawn.

## 2. **Export information**

The export information can include the following information:

**Status** The status of the export has to do with how far it has gotten in its preparation.

**File size** This is the file size of the zip file that will be downloaded Note that the file size of the contents of the zip can increas once it is downlaoded and locally extracted.

**File format** Usually ZIP

**Created at** The time and date of when the export was created.

**Created by** The user that created this export.

**Sharing** Only the user has access to private exports while all project users have access to exports that are shared with the project.

**Comment** The comment that was included with this export.

## 3. **Revisions included in this export**

The revisions list contains a list of all model revisions in the export. This is what an included revision card can look like.

![](https://raw.githubusercontent.com/catenda/help-center/main/images/3ajw00qp/02-revisions-included-in-this-export.png)

Click on the included revision card to open the [included revisions contents page](#h_e68c2c044b).

> **Note:** Revisions from models that have since been removed are not displayed in this list.

The included revision row can include the following information:

**Model name** The name of the model that the revision belongs to.

**Revision number** The nth revision in the model

**2D, 3D and download button** Click on the 2D button to load the 2D view of this revision into the 2D viewer. Click on the 3D butotn to load the 3D viewer of this revision into the 3D viewer. Click on the download button to download only this revision.

**IFC version** The version of IFC that this revision is.

**Creation date and time** The time and date the revision was created.

**Revision creator** The name of the user that created the revision.

### 3.1 **Included revision content page**

Click on an included revision in the list of included revisions in an export to see the content page for that revision. This is what the content page of an included revision can look like:

![](https://raw.githubusercontent.com/catenda/help-center/main/images/3ajw00qp/03-included-revision-content-page.png)

The included revision content page can include the following information:

**Revision number** The number of the revision

**IFC version** The version of IFC that this revision is.

**2D, 3D, document and download buttons.** Click on the 2D button to load the 2D view of this revision into the 2D viewer. Click on the 3D butotn to load the 3D viewer of this revision into the 3D viewer. Click on the document button to open the document that is linked to this model revision in the folder structure on the documents page. Click on the download button to download only this revision.

**Created at** The date and time of when this revision was created.

**Created by** The user that created this revision.

**Model** The model that this revision is in.

**Comment** The comment that goes along with this revision.
