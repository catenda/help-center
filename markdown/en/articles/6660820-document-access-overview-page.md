# Document access overview page

> An overview of the access rights of users and teams

The document access overview can be found by clicking on the access overview button in the [access control menu](https://support.catenda.com/en/articles/7831371-document-settings-page#h_98a4be4ac9) of the [document settings page](https://support.catenda.com/en/articles/7831371-document-settings-page). With the document access overview a general overview of the access rights of users and teams in the documents table can be found. _Access required:_ Administrator

This is what the access overview page can look like:

![](https://raw.githubusercontent.com/catenda/help-center/main/images/54j7njww/01-intro.png)

## 1. **Actions in the document access overview**

This is what the document access overview actions can look like:

![](https://raw.githubusercontent.com/catenda/help-center/main/images/54j7njww/02-actions-in-the-document-access-overview.png)

### 1.1 **Export to excel**

Click on the export to excel action to export the columns that are enabled in the access overview to excel.

## 2. **Generating the access overview**

when the document access overview page is opened for the first time in a project it can look something like this:

![](https://raw.githubusercontent.com/catenda/help-center/main/images/54j7njww/03-generating-the-access-overview.png)

A dialogue appears towards the bottom left of the page that shows that the access overview has started generating. Other pages can be navigated to while keeping this dialogue open to see the progress or it can be closed, the generation of the overview will continue either way.

After having navigated to a different page and the document access overview finishes generating the dialogue can look something like this:

![](https://raw.githubusercontent.com/catenda/help-center/main/images/54j7njww/04-generating-the-access-overview.png)

### 2.1 **Generation completed**

Towards the top of the generated document access overview the date and time it was generated along with a refresh button can be seen.

![](https://raw.githubusercontent.com/catenda/help-center/main/images/54j7njww/05-generation-completed.png)

The access that members and teams have to the documents are available at the time of the generation of the access overview is displayed. This way an administrator can keep an overview while access settings are rapidly being changed, for example in the startup phase of a project.

> **Note:** The generated access overview only shows the situation at the time of generation and is not dynamic. To get the latest changes anew overview has to be generated.

### 2.2 **Regenerate access overview**

When the access overview was generated more than two weeks ago, the message will turn orange indicating that it might be outdated.

![](https://raw.githubusercontent.com/catenda/help-center/main/images/54j7njww/06-regenerate-access-overview.png)

Click on the regenerate access overview action to get the latest version of the documents and the access in the project.

## 3. **Access overview table**

It is only possible to see the access for folders in the document tree. To see the access for specific document, select the document and see the [access menu](https://support.catenda.com/en/articles/8302244-right-menu-of-a-revision-revision-info#h_8525de97c2) in the [right menu of a document revision](https://support.catenda.com/en/articles/8302244-right-menu-of-a-revision-revision-info). Click [here](https://support.catenda.com/en/articles/11748020-tables-on-catenda) to read more about how to work with tables on Catenda.

### 3.1 **Filtering in the access overview table**

This is what the filter bar of the document access overview page can look like:

![](https://raw.githubusercontent.com/catenda/help-center/main/images/54j7njww/07-filtering-in-the-access-overview-table.png)

Write the name of a folder to filter on folders with that name.

> **Note:** Filtering only works when folders are retracted with the minus button in the header row of the folders column.

**Folders with own ACL** Click on folders with own ACL to only see folders for wich specific ACL is configured.

### 3.2 **Column dropdown**

Activate/deactivate users or/and teams by customizing your table. Change the display order of users and teams...

![](https://raw.githubusercontent.com/catenda/help-center/main/images/54j7njww/08-column-dropdown.png)

### 3.3 **Folders and access inherited from**

This is what the folders and access inherited from columns can look like:

![](https://raw.githubusercontent.com/catenda/help-center/main/images/54j7njww/09-folders-and-access-inherited-from.png)

Scroll down the tree of your documents.

**Header row** Click on the plus and the minus in the header row to expand and collaps the documents tree.

**Row hover** Hover over the row of a folder to see a link icon appear. Click on the link icon to open that folder in the documents table.

**Lock icon** The lock icon means that there is specific access that is configured for this folder.

**Arrow icon** The aarow icon means that this folder is inheriting access from its parent folder.

### 3.4 **User and team columns**

See the access rights of each user and team for each folder in the tree. This is what the access rights table can look like:

![](https://raw.githubusercontent.com/catenda/help-center/main/images/54j7njww/10-user-and-team-columns.png)

**Header row** See if the column is a member, administrator or team column by looking at the icon in the header row. Member icon - 👤 Adminsitrator - 👤 but then in orange Team - 👥

Click on the name or icon of a member to see which teams they are part of. Click on the name or icon of a team to see which members that are part of the team.

**Row content** Access can either be set to No access, read, write or full access. Click on the access setting to get more information about the settings that are configured.
