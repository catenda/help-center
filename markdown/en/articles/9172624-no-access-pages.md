# No access pages

There are several ways to deny access to places on Catenda Hub. Users who are only used to seeing pages they have access to might be surprised or confused when they attempt to follow a link that they expect to contain content. In this support article you will be able to find screenshots of what you can expect a page to look like when you do not have access to the content on the page you are trying to access. If you try to follow a link to a place on Catenda Hub where you do not have access to, the page will load, but you will not be able to see its contents.

### 3.1 **No access to topic boards on the boards page**

### 3.3 **Non existant topic board**

[https://hub.catenda.com/project/xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx/issues?board=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx](https://hub.catenda.com/project/xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx/issues?board=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx) This is what the topics page can look like when the user has tried to access a topic board that does not or no longer exists.

![](https://raw.githubusercontent.com/catenda/help-center/main/images/qewb0f5x/06-non-existant-topic-board.png)

**Load default topic board** Click on the load default topic board button to load the first topic board you have access to in a project.

_No access to default topic board_ Without access to the first topic board in the list of active topic boards in the project the load default topic board buttin is not displayed.

![](https://raw.githubusercontent.com/catenda/help-center/main/images/qewb0f5x/07-non-existant-topic-board.png)

To go to another topic board in this case, go to the boards page and select a topic board.

### 3.5 **No access in a topic**

[https://hub.catenda.com/project/xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx/issues/xxxx](https://hub.catenda.com/project/xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx/issues/xxxx) This is what a topic can look like if access has been removed in places.

![](https://raw.githubusercontent.com/catenda/help-center/main/images/qewb0f5x/09-no-access-in-a-topic.png)

**Assigned and Requested by** If the assignee or requestee does not have access to the topic board you will see (no access) in brackets behind their username. If the user does not exist in the project you will see their email address in brackets instead of their username

**Linking topics with #** If a topic from a board you do not have access to is linked with # in either the description or a comment you will see # with the topic number but not the title of the topic. The topic will then not show up for you in the linked topics in the right information menu.

**No access to linked documents** If you do not have access to a linked document you will see a question mark in the documents part of the right information menu.

**No access to objects** If you do not have access to the objects that are linked to the topic you will see there are some objects linked but they will be shown as _Unknown_

_Snapshot - No model access_ The snapshot contains an image from when it was added. This image does not update so it might be outdated.

![](https://raw.githubusercontent.com/catenda/help-center/main/images/qewb0f5x/10-no-access-in-a-topic.png)

If you do not have access to any models in the snapshot, you will not be able to play it as the play button has been grayed out. If you do not have access to a model you will not see it listed in model picker as in the screenshot of the snapshot.

### 4.3 **No document revisions**

[https://hub.catenda.com/project/xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx/libraries/xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx/items/xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx](https://hub.catenda.com/project/xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx/libraries/xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx/items/xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx) This is what the page can look like when the user has access to the document but does not see any revisions. One reason for seeing this page could be that there are no revisions. This can happen when a model is made on the models page. A document with the same name as the model and no revisions is then created and linked to the model. In the document structure the revision column will be empty and you will see "no revisions" written in the revision name column.

Another reason this can happen is if there are no public revisions in the document but there are draft revisions. If the user has read access to the document and has been denied access to read draft documents in project settings. In the document structure the revision column will be empty and you will see "x new drafts" written in the revision name column.

![](https://raw.githubusercontent.com/catenda/help-center/main/images/qewb0f5x/13-no-document-revisions.png)

### 4.5 **No access to a document in an approval**

[https://hub.catenda.com/project/xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx/libraries/xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx/approvals/xx](https://hub.catenda.com/project/xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx/libraries/xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx/approvals/xx) This is what no access in an approval can look like

![](https://raw.githubusercontent.com/catenda/help-center/main/images/qewb0f5x/16-br-br-no-access-to-a-document-in-an-approval.png)

**No access for reviewer** If a reviewer does not have access to an approval you will see (No access) behind the user. This can happen if the user is no longer part of the project.

**No access to documents in approval** If you cannot view a document, either because you do not have access or because the document is deleted, in an approval you will see a message:

**There is one file in the approval that you cannot see.** If you are able to publish the approval you will also see a button to remove the file from the request as you cannot publish files you do not have access to.
