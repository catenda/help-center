# Approvals page - Legacy

The approvals page can be found as a sub page to the [documents page](https://support.catenda.com/en/articles/8204673-documents-page) in the left navigation menu. The approvals page is hidden by default and will be visible when the status workflow has been activated in [document settings](https://support.catenda.com/en/articles/7831371-document-settings). If all [draft statuses](https://support.catenda.com/en/articles/7831371-document-settings#h_e6f3ffdbff) are archived and only published statuses are removed the page will disappear again as long as no approvals were created. Here an overview of the approval requests in the project can be found.

![](https://raw.githubusercontent.com/catenda/help-center/main/images/xc0e4bsa/01-intro.png)

On the approvals page the following can be done:

- Each approval request can be reviewed and validated by the team before a final validation decides if the revisions that are requested to be approved should be approved or not.
- Configuration of the course and outcome of an approval can be made.
- New approvals can be created.

## 1. **New item actions**

The new item actions can be found on the top right of the page.

![](https://raw.githubusercontent.com/catenda/help-center/main/images/xc0e4bsa/02-new-item-actions.png)

### 1.1 **New approval request**

Create a new approval request. Click [here](https://support.catenda.com/en/articles/12360159-creating-a-new-approval-request) to read more about creating a new approval request.

### 1.2 **New draft approval request**

Create a new approval request with drafts that remain from before the new status workflow was requested to be activated for a project. Click [here](https://support.catenda.com/en/articles/12360159-creating-a-new-approval-request#h_e06d6cce0d) to read more about creating a new draft approval request.

### 1.3 **Approval settings**

Configure how topics get created as a result of approval validation. Click [here](https://support.catenda.com/en/articles/12367293-approval-settings-page) to read more about approval settings. _Access required:_ Administrator

## 3. **Approvals list**

The information on the approvals page is spaced out in a way that makes it easy to understand at a glance, even for someone who has never seen a topic board before. Under the search bar a list view of all the [approvals](https://support.catenda.com/en/articles/8349418-approval-content) that fit the current filter can be found. This is what the approvals list can look like:

![](https://raw.githubusercontent.com/catenda/help-center/main/images/xc0e4bsa/04-approvals-list.png)

### 3.1 **Row content**

**Approval row** Opening the content of an approval row opens the approval page for that approval. Click [here](https://support.catenda.com/en/articles/8349418-approval-content) to read more about the approval page.

### 3.2 **Cards**

Each row in the approvals list contains an approval card with visual ques about the contents of each approval. Access required: One of the following regardless of administrator status:

- Member of the submitter team that was selected from the list of possible submitter teams for the selected workflow on behalf of which the submitting member submitted the approval request.
- Member of a reviewer team that is configured to review one of the steps for the selected workflow.
- Memember of the publisher team that is configured to submit the final approval for the selected workflow.

Only fields that have been configured in the approval will be displayed in its card. Some cards might therefore show more fields than others.

**Title** Approval title

**Statuses** Approval status is displayed in a box background and icon depending on the status.

**Publisher** Approval publisher If no publisher is set it will say anyone with write access.

**Due date** The due date of the approval. The calendar icon and text of the approval are: Gray - If the date if further than two weeks ahead in time. Orange - If the date if less than two weeks ahead in time. Red - If the approval has passed its due date.

**Created by** Approval creator and date and time of when the aproval was created.

**Approval number** The number of the approval
