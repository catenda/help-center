# Creating a new topic template - Legacy

The new topic template action can be found by clicking on the action menu to the right of the green plus button on the [topic templates page](https://support.catenda.com/en/articles/12367730-topic-templates-page) and the [topic template page](https://support.catenda.com/en/articles/12371549-topic-template-page). The action can also be found in the topic templates dropdown in the topic template are of the document [approval settings page](https://support.catenda.com/en/articles/12367293-approval-settings-page). _Access required:_ Administrator

This is what the new topic template dialogue can look like:

![](https://raw.githubusercontent.com/catenda/help-center/main/images/611mtt5t/01-intro.png)

## 1. **Naming**

A topic template has to have at least a template name and topic template title to be submitted.

### 1.1 **Template name**

The name of the topic template

### 1.2 **Title**

The title the topic will have By default it is set to receive the name of the approval request and the name of the document whose validation outcome resulted in the creation of this topic.

## 2. **Topic board**

Select one of the topic boards in the project that topics will be created in based on this template. To choose a new board, go to the boards page and create a new topic boards that topics will be created in.

### 2.1 **Status**

Pick from the list of available statuses in the selected topic board to configure the status that topics generated based on this template will have

### 2.2 **Type**

Pick from the list of available statuses in the selected topic board to configure the type that topics generated based on this template will have

## 3. **Due date and milestone**

The due date and milestone that are applied to the topic that will be created.

## 4. **Responsibility**

### 4.1 **Assigned to**

The member that is assigned to the topic that will be created. By default this is set to be the publisher of the approval. Choose between one of the following:

![](https://raw.githubusercontent.com/catenda/help-center/main/images/611mtt5t/02-assigned-to.png)

### 4.2 **Publisher**

The publisher of the approval

**File uploader** The uploader of the revision whose validation outcome resulted in the creation of this topic.

**Document owner** The member that created the document that the revision was in whose validation outcome resulted in the creation of this topic.

**Unassigned** No assignee

**Set me** The member configuring this template

**Assignee dropdown** Click on the assignee dropdown to select a specific member or team that will be assigned.

### 4.3 **Requested by**

The member that is requesting the topic. By default this is set to be the uploader of the revision whose validation outcome resulted in the creation of this topic.

![](https://raw.githubusercontent.com/catenda/help-center/main/images/611mtt5t/03-requested-by.png)

The requested by options are the same as in the assigned to field.

## 5. **Labels**

The labels that are applied to the topic upon creation.

## 6. **Description**

The description that the generated topic will have. By default the generated topic will include a link to the approval request as well as a link to the document revision in the approval request.

## 7. **Submit**

A topic template has to have at least a template name and topic template title to be submitted.

## 8. **New validation workflow**

It is not possible to create a topic template for projects starting out with the new validation workflow.
