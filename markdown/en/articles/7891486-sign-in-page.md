# Sign in page

When you go to [hub.catenda.com/signin](http://hub.catenda.com/signin) you will be presented with this page:

![](https://raw.githubusercontent.com/catenda/help-center/main/images/z7whdt00/01-intro.png)

## 1. **Email address**

Insert an email address that is connected to an existing account. If an account for an email does not exist yet, sign up for a new account instead. Click [here](https://hub.catenda.com/signup) to sign up for a new account.

### 1.1 **Single Sign On accounts**

Companies that own Catenda projects can configure SSO for Catenda users from their organization. Single Sign On (SSO) can be configured in two ways:

- Whitelist - Specific users from a domain
- Blacklist - All users from a domain except thos on a specified list

If the email address that is inserted is recognized as SSO-enabled account the password field will disappear and the following is displayed insted: This account uses SSO. Click on continue to sign in.

![](https://raw.githubusercontent.com/catenda/help-center/main/images/z7whdt00/02-single-sign-on-accounts.png)

Clicking continue to be redirected to the sign-in page of the organization. SSO users do not have an MFA requirement on projects from their organization where MFA is required.

> **Note:** Catenda Support can help if users are not redirected as expected but questions about the sign in page of the organization are better asked to the IT-department of that organization.

If the wrong email address was inserted simply change the address and the password field becomes available again.

## 2. **Password**

Here the password that matches for the account registered under the provided email address can be inserted. Click [here](https://support.catenda.com/en/articles/10798891-sign-in-and-passwords) for password troubleshooting.

## 3. **Sign in**

Click on sign in to verify the inserted credentials. After successfully completing the verification process users are redirected to the [projects page](https://support.catenda.com/en/articles/4670260-projects-page).

## 4. **MFA**

If an MFA code has been configured for an account, users are first asked to insert the MFA code from the authentication application on the device where they registered the code with their account. If there is a problem with the code or the device where the code was on has gone missing, please ask the project owner that has the main contact with Catenda (typically an administrator in the project) to request for the Authentication Code of your acount to be reset via the support bubble on this page, on Catenda or at [support@catenda.com](mailto:support@catenda.com)

## 5. **I forgot my** **password**

Enter your email address [here](https://hub.catenda.com/forgot-password) to reset your password. If you do not receive an email within 2 minutes please contact support with the support bubble on this page, on Catenda or at [support@catenda.com](mailto:support@catenda.com)

## 6. **Inviting new users**

By clicking on [sign up](https://hub.catenda.com/signup) towards the top of the sign in page users are able to create a free account even without being invited to a project. After finishing the account registration steps these users will find their account empty and without projects.

**Accepting an invitation and creating a new account** New users that do not have an account yet can be invited to a Catenda project via email by a project administrator or by an organization owner. When a user that does not have an account yet has received an invitation email and clicks on the link the account registration page will open and they will be asked to follow the account registration steps. Upon completing the steps the project invitation can be accepted if it is still valid and the user can become part of the project automatically.

**Accepting an invitation with an already existing account** Users that have previously created an account that click on the email invitation link and have not signed in yet are asked to sign in with their credentials before they are asked if they want to accept the project invitation.

If the account was created before the invitation was sent users can additionally sign in with their registered credentials and click on the project invitation notification to accept the invitation and become part of a project.

**Adding new users to project without invitation** Users belonging to an organization that owns Catenda projects may find that their account has been added to one of the organizations projects by an organization owner.
